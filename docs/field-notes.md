# Field Notes

The useful part of this repository is the small rule set around index fit and constraint risk.

The domain cases cover `index fit`, `join width`, `constraint risk`, and `plan drift`. They sit beside the smaller starter fixture so the project has both a compact scoring check and a domain-flavored review check.

The model rewards signal and confidence, then pulls back for drag. In this fixture, `edge` shows where that pullback matters most.

The point is not to make the repository bigger. The point is to make the important judgment testable.
